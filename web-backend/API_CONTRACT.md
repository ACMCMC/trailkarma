## Endpoints for Aldan (Android App → Backend)

Aldan's app calls these two endpoints to upload data after network is restored.

---

### POST /sync/location

**Purpose**: Upload the hiker's latest GPS location after regaining network.

**Request Body**:
```json
{
  "user_id": "user_001",
  "lat": 37.123456,
  "lng": -118.456789,
  "timestamp": "2026-04-18T10:00:00Z"
}
```

| Field | Type | Description |
|-------|------|-------------|
| user_id | string | Unique hiker identifier |
| lat | number | Latitude |
| lng | number | Longitude |
| timestamp | string (ISO 8601) | Time the location was recorded, e.g. `2026-04-18T10:00:00Z` |

**Response (success)**:
```json
{
  "ok": true
}
```

---

### POST /sync/report

**Purpose**: Upload a hazard, water, or species report from the hiker.

**Request Body**:
```json
{
  "user_id": "user_001",
  "type": "hazard",
  "lat": 37.123456,
  "lng": -118.456789,
  "title": "Snake on trail",
  "description": "Rattlesnake spotted near mile marker 42",
  "source": "ble_relay"
}
```

| Field | Type | Description |
|-------|------|-------------|
| user_id | string | ID of the hiker submitting the report |
| type | string | Report type: `hazard` / `water` / `species` |
| lat | number | Latitude of the report location |
| lng | number | Longitude of the report location |
| title | string | Short title |
| description | string | Full description |
| source | string | `direct` (uploaded by original hiker) / `ble_relay` (relayed via BLE) |

**Response (success)**:
```json
{
  "ok": true,
  "report_id": "abc123"
}
```

## Endpoints for Qian (Data Layer → Backend)

Qian's data is served through this endpoint to the frontend and app.

---

### GET /trail/context

**Purpose**: Return environmental context (species, weather, water, route segment) for a given location.

**Request**: GET with query parameters.

**Example**:
```
GET /trail/context?lat=37.123456&lng=-118.456789
```

**Expected Response**:
```json
{
  "likely_species": ["rattlesnake", "black bear", "mule deer"],
  "weather": {
    "condition": "sunny",
    "temp_f": 72,
    "wind_mph": 8
  },
  "water_sources": [
    {
      "name": "Agua Dulce Creek",
      "distance_miles": 2.3,
      "status": "flowing"
    }
  ],
  "route_segment": {
    "name": "PCT Section B",
    "mile_marker": 42.5
  }
}
```

## Endpoints for Suraj (Rewards / Karma)

After a hiker uploads a report, the backend triggers reward evaluation.

---

### POST /rewards/evaluate

**Purpose**: Evaluate whether an action earns Karma points or a collectible.

**Request Body**:
```json
{
  "user_id": "user_001",
  "action": "submit_report",
  "report_id": "abc123",
  "report_type": "hazard"
}
```

| Field | Type | Description |
|-------|------|-------------|
| user_id | string | User who performed the action |
| action | string | Action type: `submit_report` / `ble_relay` / `location_sync` |
| report_id | string | Related report ID (if applicable) |
| report_type | string | Report type (if applicable): `hazard` / `water` / `species` |

**Expected Response**:
```json
{
  "ok": true,
  "points_awarded": 50,
  "collectible": "Trail Guardian Badge",
  "status": "awarded"
}
```

## Internal Endpoints (Edith — Web Dashboard Only)

These are used by the family/friends web dashboard. No teammate integration needed.

---

### GET /hiker/:id/latest

**Purpose**: Get the latest known location of a specific hiker.

**Example**:
```
GET /hiker/user_001/latest
```

**Response**:
```json
{
  "user_id": "user_001",
  "display_name": "Test Hiker",
  "lat": 37.123456,
  "lng": -118.456789,
  "timestamp": "2026-04-18T10:00:00Z"
}
```

---

### GET /reports/nearby

**Purpose**: Get all reports near a given location, used for map display.

**Example**:
```
GET /reports/nearby?lat=37.123456&lng=-118.456789
```

**Response**:
```json
[
  {
    "report_id": "abc123",
    "user_id": "user_001",
    "type": "hazard",
    "lat": 37.124,
    "lng": -118.457,
    "title": "Snake on trail",
    "description": "Rattlesnake spotted near mile marker 42",
    "created_at": "2026-04-18T09:30:00Z",
    "source": "ble_relay"
  }
]
```
